📏 1D Arrays: Declare · Initialise · Defaults

// Three creation styles:
int[] a = new int[5];              // size 5, all elements DEFAULT (0)
int[] b = {10, 20, 30};            // array literal — size inferred (3)
int[] c = new int[]{1, 2};         // anonymous — needed when passing/reassigning

a[0] = 99;                          // index starts at 0
System.out.println(a.length);      // 5 — FIELD, no parentheses (String uses length())
System.out.println(a[4]);          // last valid index = length-1
a[5] = 1;                           // 💥 ArrayIndexOutOfBoundsException at RUNTIME
Element typeDefault value
int / byte / short / long0
double / float0.0
char'\u0000' (prints as blank)
booleanfalse
Object refs (String, etc.)null — calling a method on an unfilled cell throws NPE
Arrays are objects on the heap; the variable holds a reference. Size is fixed forever at creation — "resizing" means creating a new array. int[] a and int a[] are both legal; the first is Java style.
WRONG: arr.length()  →  RIGHT: array .length is a field; String .length() is a method; ArrayList .size(). Three different spellings — guaranteed exam distractor.

Section 3

🧮 2D Arrays

int[][] grid = new int[3][4];        // 3 rows × 4 cols, all 0
int[][] marks = {                     // literal init
    {85, 90},
    {75, 80, 95},                    // JAGGED — rows may differ in length!
    {60}
};

grid[1][2] = 7;                      // [row][col]
System.out.println(marks.length);     // 3   → number of ROWS
System.out.println(marks[1].length);  // 3   → columns in row 1

// Standard traversal:
for (int r = 0; r < marks.length; r++) {
    for (int c = 0; c < marks[r].length; c++)   // use marks[r].length — jagged-safe
        System.out.print(marks[r][c] + " ");
    System.out.println();
}

// for-each version:
for (int[] row : marks)
    for (int v : row) System.out.print(v + " ");
A Java 2D array is really an array of array references — that's why jagged rows are possible and new int[3][] (rows only, columns later) is legal.
Arrays.toString on 2D prints references — use Arrays.deepToString(marks); likewise Arrays.deepEquals for comparison.

📚 ArrayList Basics

import java.util.ArrayList;

ArrayList<String> names = new ArrayList<>();    // <> diamond — type inferred
ArrayList<Integer> nums = new ArrayList<>();    // WRAPPER — never ArrayList<int>
nums.add(5);                                     // autoboxing: int → Integer
PropertyDetail
Dynamic sizeGrows/shrinks automatically — the whole point vs arrays
Generics <T>Compile-time type safety: names.add(42) won't compile
Objects onlyPrimitives go in via wrapper + autoboxing
Ordered + indexedKeeps insertion order; get(i) by position
Duplicates & nullBoth allowed
Raw new ArrayList() (no generics) compiles with warnings and accepts anything — exam code uses it to smuggle mixed types that later explode in a cast.

✏️ ArrayList CRUD Operations

ArrayList<String> list = new ArrayList<>();
list.add("OCI");                 // C — append           [OCI]
list.add(0, "Java");             // C — insert at index  [Java, OCI]  (shifts right)
list.get(1);                     // R — "OCI"
list.set(1, "PLSQL");            // U — replace, returns OLD value
list.remove(0);                  // D — by INDEX          [PLSQL] (shifts left)
list.remove("PLSQL");            // D — by OBJECT, returns boolean

list.size();  list.contains("OCI");  list.indexOf("OCI");
list.isEmpty();  list.clear();
MethodReturnsGotcha
add(e) / add(i,e)boolean / voidInsert shifts subsequent elements right
get(i)elementBad index → IndexOutOfBoundsException
set(i,e)old elementReplaces; does NOT insert
remove(i) / remove(obj)removed elem / booleanSee trap ↓
size()intMethod — vs array .length field
remove overload ambush: on ArrayList<Integer>, list.remove(2) removes INDEX 2 (int matches remove(int) first), not the value 2. To remove the value: list.remove(Integer.valueOf(2)).
Section 6

🚶 Iterating with Iterator

import java.util.Iterator;

Iterator<String> it = list.iterator();
while (it.hasNext()) {           // 1. anything left?
    String s = it.next();         // 2. take it & advance
    if (s.startsWith("X"))
        it.remove();              // 3. SAFE removal during iteration
}
Way to iterateCan remove while looping?
for-each❌ list.remove(...) inside → ConcurrentModificationException
Iterator + it.remove()✅ the sanctioned way
Classic indexed for⚠️ works but indexes shift after each removal (iterate backwards to stay sane)
Calling it.next() when hasNext() is false → NoSuchElementException. Calling it.remove() before any next() → IllegalStateException. The hasNext→next→(remove) rhythm is mandatory.
for-each is actually compiled down to an Iterator behind the scenes — that's exactly why structural modification mid-loop triggers ConcurrentModificationException.

⚔️ Array vs ArrayList

ArrayArrayList
SizeFixed at creationDynamic — grows automatically
Primitives✅ int[] direct❌ wrappers only (autoboxing cost)
Length/size.length field.size() method
Accessa[i]list.get(i)
Insert/delete middleManual shiftingBuilt-in (shifts internally)
Multi-dimensional✅ int[][]Nested lists (clunkier)
PerformanceFastest, least memorySlight overhead (object wrapper, resize)
Utilitiesjava.util.ArraysRich instance methods + Collections class
Rule of thumb: known fixed count of primitives → array; anything that grows/shrinks or needs rich operations → ArrayList. Internally ArrayList IS backed by an array it silently replaces when full (~1.5× growth).
