Section 1

🔁 The for Loop

for (int i = 0; i < 5; i++) {   // init → condition → body → update → condition → …
    System.out.println(i);      // 0 1 2 3 4
}
// i is OUT OF SCOPE here — declared in the for header, dies with the loop

2 · Condition

VariantExampleNote
All parts optionalfor(;;) { }Infinite loop — compiles fine
Multiple countersfor(int i=0, j=10; i<j; i++, j--)Comma-separated, same type in init
Countdownfor(int i=5; i>0; i--)5 4 3 2 1
Stepfor(int i=0; i<=20; i+=5)0 5 10 15 20
Exam counting drill: for(int i=0; i<5; i++) runs 5 times; i<=5 runs 6. Off-by-one answers are always among the options.
WRONG: for(int i=0; i<5; i++); then a block "loops 5 times"  →  RIGHT: the semicolon is the entire (empty) body; the block runs once, after the loop.
Section 2

🎯 Enhanced for-each Loop

int[] marks = {85, 92, 78};
for (int m : marks) {              // "for each m in marks"
    System.out.println(m);
}

ArrayList<String> names = new ArrayList<>();
for (String n : names) { /* works on any Iterable */ }
Classic forfor-each
Index access✅ i available❌ no index
Modify elements✅ arr[i] = …❌ loop variable is a COPY — assigning it changes nothing in the array
Reverse / skip traversal✅ full control❌ forward only, every element
ReadabilityVerboseClean — "for every element"
Works onAnything indexedArrays + any Iterable
for (int m : marks) m = 0; leaves the array untouched — m is a copy of each element. To zero the array you need the indexed for. (For object elements, calling a mutating method on the copy DOES affect the shared object.)
Section 3

⏳ while Loop

int i = 0;                       // init OUTSIDE
while (i < 5) {                  // condition checked BEFORE body — may run 0 times
    System.out.println(i);
    i++;                          // update INSIDE — forget this = infinite loop
}

// Sentinel pattern — loop until a stop value:
Scanner sc = new Scanner(System.in);
int n = sc.nextInt();
while (n != -1) {                // -1 is the sentinel
    total += n;
    n = sc.nextInt();
}
Choose while when the number of iterations is unknown in advance (user input, searching, polling). Choose for when you can count.
while (i < 5); with a semicolon = empty body + i never changes = infinite loop that appears to hang. Same stray-semicolon family as if/for.
Section 4

🔂 do-while Loop

int choice;
do {
    System.out.println("1.Deposit 2.Withdraw 3.Exit");
    choice = sc.nextInt();
    // … handle choice …
} while (choice != 3);           // ← SEMICOLON REQUIRED here
whiledo-while
Condition checkedBefore body (entry-controlled)After body (exit-controlled)
Minimum runs01 — always runs at least once
Classic useUnknown-count processingMenus, input validation ("ask, then re-ask")
Trailing ;NoYes, after while(cond) — omitting it = compile error
Exam check: int i=10; do { print(i); i++; } while (i < 5); prints 10 — the body executed once before the (false) condition was ever tested. A plain while would print nothing.
Section 5

🛑 break

for (int i = 1; i <= 10; i++) {
    if (i == 4) break;             // leave the loop ENTIRELY
    System.out.print(i + " ");      // 1 2 3
}

// Labeled break — escape NESTED loops in one jump:
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i * j == 4) break outer;  // exits BOTH loops
        System.out.println(i + "," + j);
    }
}
FactDetail
Plain breakExits only the innermost enclosing loop or switch
Labeled breakExits the labeled loop — Java's answer to goto for nested escapes
Outside loop/switchCompile error: "break outside switch or loop"
Section 6

⏭️ continue

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;          // skip the REST of this pass only
    System.out.print(i + " ");      // 1 2 4 5
}
breakcontinue
EffectLoop is OVERTHIS pass is over; loop continues
In a forJumps past the loopJumps to the update (i++), then condition
In a whileJumps past the loopJumps straight to the condition
continue in a while loop placed BEFORE the manual increment skips the increment → infinite loop: while(i<5){ if(i==3) continue; i++; } spins forever at i=3. In a for, the update always runs — that's why for is safer with continue.

Nested loop pattern — the star pyramid

for (int row = 1; row <= 4; row++) {      // outer = rows
    for (int col = 1; col <= row; col++)   // inner = columns per row
        System.out.print("*");
    System.out.println();
}
/*  *
    **
    ***
    ****   — inner loop runs 1+2+3+4 = 10 times total */
Total inner-body executions in nested loops = product (independent bounds) or a triangular sum (dependent bound like col ≤ row). Exams ask "how many times does * print?"

// #5 solution core — reverse digits:
int n = 1234, rev = 0;
while (n > 0) {
    rev = rev * 10 + n % 10;   // shift left, append last digit
    n /= 10;                    // drop last digit
}
System.out.println(rev);        // 4321
The %10 / ÷10 digit-extraction idiom powers exercises 5 and 6 and appears in certification questions disguised as "what does this loop print".
