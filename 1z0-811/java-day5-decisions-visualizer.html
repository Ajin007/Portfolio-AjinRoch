Section 1

🔱 if · if-else · else-if Ladder

if (marks >= 90) {                 // condition MUST be boolean — if(1) is a compile error
    grade = "A";
} else if (marks >= 75) {          // checked only if the first was false
    grade = "B";
} else if (marks >= 60) {
    grade = "C";
} else {                           // optional catch-all
    grade = "F";
}
RuleDetail
Condition typeStrictly boolean. if (x = 5) fails… but if (flag = true) COMPILES (assignment yields boolean) — evil exam favourite.
Ladder orderConditions checked top-down; first true wins, rest skipped. Order ranges high→low or you'll never reach the tighter bands.
Braces optionalWithout {} only the FIRST statement belongs to the if — indentation is a lie the compiler ignores.
Dangling else: an else binds to the nearest unmatched if, not the one your indentation suggests. Braces eliminate the ambiguity — always use them.
WRONG: if (x == 5); { … } "conditionally runs the block"  →  RIGHT: the semicolon is an empty body; the block always runs. Spot stray semicolons after if/for/while.
Section 2

🪆 Nested if

if (age >= 18) {
    if (hasVoterId) {                          // inner check runs only when outer is true
        System.out.println("Eligible to vote");
    } else {
        System.out.println("Apply for voter ID");
    }
} else {
    System.out.println("Minor — not eligible");
}

// Often flattenable with &&:
if (age >= 18 && hasVoterId) { /* eligible */ }
Nested if vs &&: use nesting when each branch needs a different else message; use && when only the combined-true path matters. Exams trace which println fires for given inputs — draw the decision tree.
Section 3

🔀 switch Statement & Fall-through

int day = 3;
switch (day) {
    case 1:  System.out.println("Mon"); break;
    case 2:  System.out.println("Tue"); break;
    case 3:                                     // no break →
    case 4:  System.out.println("Midweek"); break;  // 3 FALLS THROUGH to 4
    default: System.out.println("Other");       // default can sit ANYWHERE
}
RuleDetail
Allowed typesbyte, short, char, int, their wrappers, String (Java 7+), enum. NOT long, float, double, boolean.
Case valuesCompile-time constants, unique, assignable to switch type. Duplicate case = compile error.
Fall-throughWithout break, execution continues into the NEXT case body — including default.
defaultOptional; runs when nothing matches; position doesn't matter (fall-through still applies).
// Modern switch EXPRESSION (Java 14+) — arrow syntax, no fall-through, returns a value:
String type = switch (day) {
    case 1, 7 -> "Weekend";
    case 2, 3, 4, 5, 6 -> "Weekday";
    default -> "Invalid";
};
Exam pattern: give a matching case with missing breaks and ask for output — the answer includes every case body from the match downward until a break or the closing brace.
WRONG: "default must be last."  →  RIGHT: default can appear anywhere; if it matches (no other case did) and lacks break, it falls through to whatever follows it.
Section 4

🔢 Comparing Primitives (==)

int a = 5; long b = 5L;
a == b                     // true — a is PROMOTED to long before comparing

char c = 'A';
c == 65                    // true — char promoted to int

double x = 0.1 + 0.2;
x == 0.3                   // FALSE! binary floating-point → 0.30000000000000004
Math.abs(x - 0.3) < 1e-9   // ✅ epsilon comparison for doubles
ComparingCorrect tool
int, char, boolean, long…== compares values directly ✅
double / floatEpsilon difference, or BigDecimal for money
Different numeric typesSmaller side auto-promotes, then values compared
For primitives == is value comparison and always safe (except floating-point precision). The == danger zone begins with objects — next card.
Section 5

🧿 Comparing Objects: equals() & instanceof

String s1 = new String("Java");
String s2 = new String("Java");
s1 == s2            // false — two heap objects
s1.equals(s2)       // true  — content match

// instanceof — safe type check (null is NEVER an instance of anything):
Object o = "text";
if (o instanceof String) {
    String s = (String) o;              // classic: check then cast
}
if (o instanceof String str) {          // Java 16+ PATTERN MATCHING: check + cast + bind
    System.out.println(str.length());
}
FactDetail
Default equals()Object.equals() is identity (same as ==). String/Integer/ArrayList override it for content. Your own classes compare references until YOU override equals().
null instanceof XAlways false — never throws. Safe pre-cast guard.
x.equals(null)Contract says return false; calling equals ON null throws NPE.
A custom class without an equals() override: two objects with identical field values are not equal — new Employee("A").equals(new Employee("A")) → false. Sets/Maps then treat them as distinct entries.

📝 Quiz 1 — Days 1–4 Review

Classroom checkpoint — 10 rapid questions. Answers at the bottom (don't peek).

#Question
1Which component actually executes bytecode — JDK, JRE, or JVM?
2javac Hello fails. Why?
3Where must the package statement appear?
4Is int 2count; legal? Is int _count;?
5What is printed: System.out.println(1 + 2 + "3" + 4);
6byte b = 10; b = b + 1; — compiles? What about b += 1;?
7Integer a=127,b=127; a==b? And with 128?
8Output of "Java".substring(1,3)?
9String s="hi"; s.toUpperCase(); System.out.println(s); prints?
10Range of Math.random() — is 1.0 possible?

Answer key

1JVM — JDK compiles, JRE packages libraries+JVM, JVM executes.
2javac needs the .java extension; java (the launcher) takes the bare class name.
3First non-comment statement of the file; at most one.
42count ❌ (digit start); _count ✅ (single "_" alone is illegal).
5"334" — 1+2 arithmetic → 3, then string concat glues "3" and 4.
6b=b+1 ❌ (b+1 is int, needs cast); b+=1 ✅ hidden cast.
7127 → true (Integer cache -128..127); 128 → false; equals() → true both times.
8"av" — begin inclusive, end exclusive.
9"hi" — result of toUpperCase() was discarded; Strings are immutable.
100.0 ≤ x < 1.0 — 1.0 is never returned.
Score 8+/10 → ready for loops (Day 6). Below 6 → replay the Day 3 promotion rules and Day 4 immutability cards before moving on.
