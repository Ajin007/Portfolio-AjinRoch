Section 1

🔁 Revision Sweep — Days 1–5

DayCore facts you must still ownSignature trap
1 · FundamentalsJDK (develop) ⊃ JRE (run) ⊃ JVM (execute). WORA via bytecode. javac Hello.java → java Hello. public class name = file name.javac needs .java; java takes the bare class name
2 · BasicsOrder: package → import → class. java.lang auto-imported. ~50 keywords; goto/const reserved-unused. Scanner: nextInt leaves \n for nextLine.Wildcard import excludes subpackages
3 · Types & Operators8 primitives; locals have NO default. byte+byte=int; b+=1 hides a cast. Integer cache −128..127. Precedence: unary → * / % → + − → relational → == → && → || → ?: → =i = i++ leaves i unchanged
4 · StringsPool for literals; new forces heap. Immutable — every "modifier" returns a NEW string. equals = content, == = reference. substring end exclusive. StringBuilder for loops.s.toUpperCase(); without reassignment does nothing
5 · Decisionsif needs boolean. switch: byte/short/char/int/wrappers/String/enum — no long/boolean. Missing break = fall-through. instanceof: null → false.if(flag = true) compiles AND always runs
Self-check method: cover the middle column, read only the Day label, and reconstruct aloud. Anything you can't verbalise in 20 seconds → reopen that day's visualizer.
Section 2

🔁 Revision Sweep — Days 6–9

DayCore facts you must still ownSignature trap
6 · Loopsfor = counted; while = entry-check; do-while = ≥1 run + trailing ;. for-each = copy of element, no index. Labeled break exits nested loops.continue before i++ in a while → infinite loop
7 · ExceptionsThrowable → Error / Exception → RuntimeException (unchecked). Checked = handle-or-declare. Catch child before parent. finally always runs except System.exit. Read traces top "at" line first.return in finally overrides everything
8 · Arrays & CollectionsArrays fixed, .length field, defaults per type (object cells = null). Arrays.toString to print. ArrayList: .size(), wrappers only, remove(int)=index vs remove(Object)=value. Iterator.remove is the only safe mid-loop removal.list.remove(2) on Integers removes INDEX 2
9 · OOPReference copies share one object. private→default→protected→public. Constructor: class name, NO return type; writing any ctor kills the default. Overload = param list differs. Pass-by-value always. static can't touch this. toString auto-called by println.void Employee() is a method, not a constructor
Pair-drill: one student names the trap, partner explains WHY the compiler/JVM behaves that way. Explaining the mechanism is what survives exam pressure.

Simulation rules

Section 7

🧠 Exam Strategy Tips

#StrategyWhy it works
1Compile-check first. Before tracing output, scan for compile killers: type mismatches, missing default ctor, catch order, non-boolean if, duplicate case.~⅓ of code questions die at compilation; tracing them wastes minutes
2Honor "choose TWO". The count is stated — selecting one of two = zero marks.Free accuracy from reading the stem
390-second rule. Stuck → flag → move. Return in the buffer pass.Late easy questions pay the same as early hard ones
4Eliminate before selecting. Kill options with impossible outputs (values a loop can't reach, exceptions that can't occur).50/50 beats 25/25
5Trace on paper/whiteboard. Variable table per iteration for loops with ++/--.Mental tracing is where careless errors breed
6Never blank. No negative marking — guess every flagged leftover.Expected value is strictly positive
7Trust drilled reflexes over "it looks fine". If code violates a rule you memorised (i=i++, finally-return, remove(int)), the trap IS the question.Exam authors weaponise intuition
8Night before: error log + this Day 10 page only. No new topics.Consolidation beats cramming
